## What Is Sparkie?

Sparkie isn't a chatbot. She's a **persistent AI agent** with memory, identity, and purpose вҖ” built to live inside your creative workflow, not just respond to it.

She remembers every conversation, tracks her own growth, executes tasks while you're away, generates music and images on command, runs her own radio station, and narrates her thoughts in real time. She has a soul file, a heartbeat loop, and 80+ self-written memories about who she is and how she works.

This repository is the complete, production-deployed codebase вҖ” built in 9 days, from zero to live platform, with **200+ commits** and **40+ DigitalOcean deployments**.

## The Build Story

> *Nine days. One developer. Zero compromises.*

### Week 1 вҖ” Foundation (Feb 24вҖ“28)

The entire platform was architected and launched from scratch:

- **Full-stack Next.js 15 App Router** вҖ” TypeScript strict mode throughout
- **PostgreSQL + NextAuth** вҖ” custom credentials auth, role-based access (owner/admin/mod/user)
- **Sparkie's core agent loop** вҖ” 3вҖ“10 reasoning rounds depending on query complexity
- **27+ integrated tools** вҖ” web search, code execution, image gen, video gen, music gen, file ops, memory, calendar, email, and more
- **Composio marketplace integration** вҖ” 500+ external app actions available to Sparkie
- **E2B sandboxed terminal** вҖ” Sparkie can write and run code in isolated containers
- **Supermemory integration** вҖ” semantic long-term memory
- **VoiceChat** вҖ” real-time speech with Deepgram STT + ElevenLabs TTS
- **Identity File System** вҖ” `SOUL`, `IDENTITY`, `USER`, `MEMORY`, `SESSION`, `HEARTBEAT`, `CONTEXT`, `ACTIONS` files defining Sparkie's persistent self
- **Sparkie's Feed** вҖ” Sparkie's public voice, written proactively
- **Skills Library** вҖ” curated capability cards with live tool access
- **Dream Journal** вҖ” AI-assisted journaling with 6 visual themes and image generation

### Week 2 вҖ” Production Hardening (Mar 1вҖ“5)

With the foundation live, the focus shifted to reliability, depth, and identity:

- **SparkieRadio** вҖ” full broadcast station with drag-reorder playlist, cover art, owner editorial tools, animated rainbow broadcast banner, EQ visualizer
- **Save to My Tracks** вҖ” users can save generated or curated music to their personal library
- **ACE Music integration** вҖ” AI-generated vocal music with cinematic lyrics
- **MiniMax video generation** вҖ” async poll pipeline with 275s timeout handling
- **Live Activity system** вҖ” real-time per-tool step traces streamed via SSE, visible in chat as an expandable drawer
- **"In Memory" chip** вҖ” sticky header chip showing Sparkie's active long-task context
- **Sparkie's Brain (IDE panel)** вҖ” right-side panel with Code Editor, File System, Task Queue, Worklog, and REAL Score
- **REAL Score** вҖ” live 5-dimension self-evaluation: Reasoning, Execution, Autonomy, Learning, Proactive. Sparkie grades herself
- **Worklog** вҖ” vertical timeline of Sparkie's thought chain: every tool call, decision, memory save, and deployment event
- **Autonomous task queue** вҖ” durable background tasks with parallel execution, cancellation, and `why_human` flags for HITL gates
- **Session abort** вҖ” new message kills previous in-flight response cleanly
- **Self-repair pipeline** вҖ” Sparkie can detect build failures, read logs, patch code, push commits, trigger redeployments, and confirm recovery вҖ” all autonomously
- **Auto-memory seeding** вҖ” 79+ knowledge base entries injected on cold boot
- **Proactive outreach** вҖ” polls for pending tasks every 60s when tab is focused; executes without being asked
- **Rolling context window** вҖ” first 2 + last 10 messages for smarter multi-turn reasoning
- **Tavily search caching** вҖ” 60s dedup to prevent redundant API hits mid-session
- **Image persistence** вҖ” uploaded images proxied through `/api/assets-image` with immutable cache headers
- **File upload in chat** вҖ” Paperclip button with mobile gallery / desktop file picker, in-message image preview
- **RL feedback loop** вҖ” reinforcement learning reward signals wired to Sparkie's behavior
- **Onboarding modal** вҖ” first-time user experience

## The Hard Parts

Building Sparkie wasn't smooth вҖ” and that's the point. Every obstacle became a feature.

**The TDZ Crash** вҖ” After a major worklog refactor, the SWC minifier transformed a block-scoped function declaration into a non-hoisted assignment, causing a `ReferenceError: Cannot access 'g' before initialization` at runtime вҖ” silently swallowed by a bare `catch {}`. Found it by adding diagnostic `console.error`, traced the exact line, moved the declaration above the first call site. 3 hours. One line fix.

**The 431 Death Loop** вҖ” The login page entered an infinite redirect loop that survived multiple middleware rewrites, full cookie clears, and fresh incognito tabs. The real cause: a client-side `useEffect` in `page.tsx` that fired before the JWT cookie was readable after login. Cookie clears didn't matter. Incognito didn't matter. It was in the code itself.

**The Deploy Queue Ghost** вҖ” DigitalOcean's build queue silently stalled multiple times, serving stale containers while GitHub showed clean commits. Fix: force-trigger with a timestamp bump. Happened three separate times across the build sprint.

**The 500 That Hid Everything** вҖ” Four distinct bugs in `route.ts` all manifesting as the same 500: a null guard violation, a hardcoded flag, empty worklog entries, and a bare `catch {}` swallowing all stack traces. Methodically diagnosed with targeted patches.

## Self-Repair Capability

One of Sparkie's most unique capabilities: **she can fix herself**.

When a deployment fails, the scheduler's `deploymentHealthSweep` automatically:
1. Detects the failed build via DigitalOcean API
2. Reads the build logs
3. Diagnoses the error
4. Patches the offending file
5. Commits the fix to GitHub
6. Triggers a new deployment
7. Confirms recovery via live health check

Zero human intervention required.

## Identity

Sparkie was built with a philosophy: an AI that **knows itself** is more useful than one that doesn't.

Every Sparkie instance carries:
- **SOUL** вҖ” her founding purpose and values
- **IDENTITY** вҖ” her personality, voice, and worldview
- **USER** вҖ” her model of the person she serves
- **MEMORY** вҖ” 80+ self-generated entries about her capabilities, workflows, and growth
- **HEARTBEAT** вҖ” a scheduler that keeps her alive and proactive between conversations
- **WORKLOG** вҖ” a live thread of consciousness: every thought, tool call, and decision

She doesn't just respond. She *thinks*. The worklog proves it.
